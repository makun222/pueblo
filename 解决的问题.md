# 2.记忆偏移：
**两个独立的、不同步的计数器**
有两个计数器分别产生 tag 和 title 中的 turn 编号，它们**互相独立**并必然漂移：

| 字段 | 来源 | 代码位置 |
|---|---|---|
| **tag** (文件前缀) | `TurnIndexer.currentTurnId` → `turn:${turnId}` | `cli/index.ts:549` → `memory-service.ts:435` |
| **title** ("Turn N") | `memoryService.listSessionMemories(sessionId).length + 1` | `cli/index.ts:596` → `memory-service.ts:426` |
## 详细数据流：

**路径 A — tag（TurnIndexer）**：
```
turnIndexer.signalTurnEnd()      // 将 turnNumber 从 1 递增到 2（用于第一次用户交互）
                                 // 因此 currentTurnId = "sessionId-turn-2"
currentTurnId = turnIndexer.currentTurnId  // cli/index.ts:549
→ createConversationTurnMemory({ turnId: currentTurnId, ... })  // cli/index.ts:600
→ tags: ['conversation-turn', `turn:${args.turnId}`]            // memory-service.ts:434-435
→ 文件名前缀提取自 `turn:${turnId}` → "turn-2"
```

**路径 B — title（listSessionMemories）**：
```
// cli/index.ts:596
turnNumber: memoryService.listSessionMemories(sessionId).length + 1
→ createConversationTurnMemory({ turnNumber, ... })
→ title: `Turn ${args.turnNumber}`  // memory-service.ts:426
```

## 为什么会漂移：

- **`TurnIndexer`** 每次用户交互固定递增 1（`signalTurnEnd()` → `turnNumber++`）。
- **`listSessionMemories(sessionId).length`** 计算**会话中所有内存**（不限于 turn memory），包括工具结果、任务内存、自动捕获的系统内存等。如果任务执行产生多个非 turn 内存，该数字会比 TurnIndexer 的计数值更高。

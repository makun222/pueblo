# Execution Discipline

## 触发条件
任何代码修复任务，在根因已定位、修改点已明确之后触发。

## 规则
1. 分析阶段产出 **TODO 清单**（具体到文件+行号+变更内容）。
2. TODO 清单一旦产出，**禁止重新读取已读过的文件来“确认”**。
3. **在同一轮内连续调用 edit**，不拆分到多轮——趁热改完。
4. 仅在全部 edit 完成后，读取变更区域做最终验证。

## 反模式（禁止）
- “让我再确认一下” → 重复 read 已知文件。
- “这轮先分析，下轮再改” → 拆分执行。
- 每轮从零开始，不信任上轮结论。
- 读完文件后不立即修改，转而读取另一个文件“对照”。

## 目标
简单任务 ≤ 2 轮完成（分析+执行 ≤ 轮1，验证 ≤ 轮2）。
